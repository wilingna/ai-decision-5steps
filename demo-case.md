# 示范案例：某B端SaaS业务裁撤决策

## 案例背景数据

### 财务数据
| 指标 | 数值 |
|------|------|
| 2022年营收 | 8200万（+18%） |
| 2023年营收 | 9100万（+11%） |
| 2024年营收 | 9300万（+2%） |
| 总成本 | 1.25亿元 |
| 人力成本占比 | 58% |
| 毛利率 | 32% |
| 净利润 | -1800万（连续第二年亏损） |
| 现金流 | -1200万 |
| 研发投入/营收 | 38% |

### 客户数据
| 指标 | 数值 |
|------|------|
| 客户总数 | 420家 |
| 前5大客户收入占比 | 41% |
| 客户续费率 | 76% |
| 客单价 | 21万/年 |
| 新签客户增速 | 4% |
| 行业增速 | 8%-12% |
| 同行平均毛利率 | 45% |

### 决策问题
是否在未来12个月内裁撤该业务？

---

## Step 1 示范输出：行业事实摘要

```
【行业事实摘要】（B端协同/流程类SaaS）

1. 市场规模与增速：
   - 2023-2025年中国B端SaaS市场增速放缓至8%-12%区间
   - 头部集中效应加剧，中小厂商增速显著低于行业均值

2. 行业平均毛利率区间：
   - 成熟B端SaaS毛利率通常45%-65%
   - 低于35%通常意味着交付成本过高或定价偏低

3. 客户采购/回款趋势：
   - 2023-2024年企业IT预算收紧，回款周期延长
   - 大客户谈判力增强，折扣压力上升

4. 并购整合趋势：
   - 行业进入整合期，小规模SaaS被收购或关闭案例增加
   - 垂直行业SaaS相对水平SaaS更抗周期

5. AI对该行业的影响：
   - AI功能成为续约/新签关键因素
   - 部分传统SaaS被AI原生产品替代，尤其是标准化程度高的流程类产品

【与用户业务的初步对比】
- 用户增速（2%） vs 行业增速（8-12%）：严重落后
- 用户毛利率（32%） vs 行业均值（45%）：低13个百分点
- 初步信号：业务竞争力可能在下滑，需结构化分析找原因
```

---

## Step 2 示范输出：结构化梳理

```
【A. 财务与现金流事实】
- 营收：9300万，增速从18%降至2%，增速连续两年大幅下滑
- 毛利率：32%（行业均值45%，差距13pp）
- 净亏损：1800万，连续第二年亏损
- 现金流：-1200万
- 研发投入：38%营收，显著高于行业常规（通常15-25%）

【B. 增长与客户结构事实】
- 客户总数420家，客单价21万/年
- 续费率76%（健康续费率通常>85%，当前偏低）
- 前5大客户占41%（客户集中度：高风险）
- 新签增速4% vs 行业增速8-12%：市场份额在流失

【C. 成本与组织事实】
- 人力成本占总成本58% → 固定成本压力极高
- 总成本1.25亿 > 营收9300万 → 每年必然亏损
- 研发投入高但增速持续下滑 → 研发效率存疑

【D. 行业趋势映射事实】
- AI替代路径：若该SaaS是标准化流程工具，AI原生产品可能进一步抢占市场 → 续费率可能继续下滑
- 采购收紧趋势 → 新签增速可能进一步承压
- 行业整合趋势 → 若不主动，可能被动面临更差条件的退出

【事实缺口清单】（当前还缺以下信息）
1. 76%续费率中，有多少是靠折扣/降价续回来的？（真实留存质量）
2. 研发38%投入具体在做什么？是维护存量还是投AI转型？
3. 1800万亏损中，有多少是战略性投入（研发/销售扩张）？
4. 前5大客户的合同到期时间？（近期是否有流失风险）
5. 人力成本中，核心研发/销售 vs 可裁撤人员的比例？
6. 现金储备/母公司支持能力（还能支撑多久亏损）？
7. 是否有潜在收购方/战略合作方？
8. 产品AI化改造需要多少投入和时间？
```

---

## Step 3 示范输出：三路径博弈

```
【路径A：立即裁撤】

12M现金流影响：
- 方向：短期现金流大幅改善
- 区间：节省约1.2亿成本 - 裁员补偿(预估2000-4000万) - 客户违约赔偿(不确定)
- 关键变量：裁员补偿标准、现有合同违约条款

2年后状态：
- Base：完成清算，现金流回正，资源集中主业
- Downside：大客户诉讼/索赔，品牌损伤持续发酵

关键风险：
- 财务：裁员补偿可能达3000万+，短期现金压力大
- 客户：41%大客户服务中断，可能带来索赔和声誉损失
- 声誉：市场信号负面，影响公司其他业务招商/融资
- 组织：核心人才流失，若后续想重入市场成本极高

可逆性：极低 ⚠️ 一旦执行，客户关系、团队、市场信誉均难恢复

---

【路径B：缩减规模保留核心】

12M现金流影响：
- 方向：亏损收窄，但仍为负
- 区间：预计亏损降至500-900万（假设：人力成本削减40%，维持核心客户服务）
- 关键变量：能否保留核心客户续费、裁员后服务质量是否下滑

2年后状态：
- Base：业务瘦身稳定，等待行业拐点或收购机会
- Downside：客户因服务质量下滑持续流失，最终仍需关闭

关键风险：
- 客户：服务能力下降可能触发大客户流失连锁反应
- 组织：核心员工不确定性高，人才流失风险
- 战略：处于"不上不下"的尴尬状态，竞争力持续弱化

可逆性：中等——可以随时转向A或C，但每拖一个月都在消耗现金

---

【路径C：继续投资AI化转型】

12M现金流影响：
- 方向：亏损扩大
- 区间：亏损可能增至2500-3500万（假设：AI改造需追加投入1000-1500万）
- 关键变量：AI改造能否在12-18个月内带来续费率提升和新签加速

2年后状态：
- Upside：AI功能上线，续费率回升至85%+，新签增速追上行业，毛利率改善至40%+
- Downside：AI改造延期或效果不达预期，现金流危机，被迫以更差条件退出

关键风险：
- 财务：追加投入后现金流极度承压
- 技术：AI改造是否真的能解决竞争力问题？还是交付成本问题？
- 市场：行业AI原生竞品可能在同期推出，窗口期有限

可逆性：低——投入越深，退出成本越高

---

【三路径可逆性对比】
路径A：不可逆性【极高】— 客户关系、团队、声誉一旦破坏难以重建
路径B：不可逆性【中】— 保留了退出和投资的选项窗口
路径C：不可逆性【高】— 投入越深，心理成本和实际成本都让退出更难

最关键问题：路径C的前提是"AI改造能解决问题"——但如果竞争力问题根源是定价、销售效率或客户成功，AI改造并不能修复。
```

---

## Step 4 示范输出：综合评分

| | 路径A 立即裁撤 | 路径B 缩减规模 | 路径C 继续投资 |
|--|--|--|--|
| 现金流得分(40%) | 8 | 6 | 2 |
| 增长潜力得分(30%) | 1 | 3 | 8 |
| 战略价值得分(20%) | 2 | 4 | 7 |
| 风险得分(-10%) | 6 | 7 | 3 |
| **加权总分** | **4.8** | **5.1** | **5.4** |
| 决策建议等级 | 谨慎 | 可行 | 高风险可行 |

**注意**：路径C总分最高，但假设前提最脆弱。如果用户判断"AI改造解决不了根本问题"，应下调路径C的增长潜力得分。**这就是为什么最终判断权在人不在AI。**
