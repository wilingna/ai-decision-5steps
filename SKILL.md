# AI 决策五步法 Skill

## 工具分工总览

**核心原则：每个工具只做它最擅长的事。**

| 步骤 | 层次 | 工具 | Claude的角色 |
|------|------|------|------------|
| Step 1 | 信息层 | **Perplexity** | 生成定制Prompt → 用户去Perplexity执行 |
| Step 2 | 结构层 | **Claude** | ⭐ Claude亲自执行结构化分析 |
| Step 3 | 推理层 | **GPT** | 生成定制Prompt → 用户去GPT执行 |
| Step 4 | 系统层 | **GPT → Notion** | 生成两个Prompt（对比表+CSV）→ 用户导入Notion |
| Step 5 | 表达层 | **GPT → Gamma** | 生成汇报大纲Prompt → 用户去GPT执行后带去Gamma |

---

## 启动流程

### 1. 判断决策类型

**企业/业务决策**：涉及业务线、营收、团队、裁撤、投资、转型等

**个人重大决策**：读博vs工作、做不做副业、选赛道、跳槽、内容创作方向等

### 2. 收集必要信息

分两轮收集，不要一次全问：

**企业决策**

第一轮（必须）：近2-3年营收及增速、毛利率/净利润/现金流、决策问题和时间窗口

第二轮（补充）：客户数/客单价/续费率/新签增速、客户集中度、成本结构、公司当前阶段（扩张/稳健/收缩）

**个人决策**

第一轮（必须）：决策问题（越具体越好）、当前处境（职业阶段/收入/时间）、最核心的目标是什么

第二轮（补充）：能承受多长时间收入下降、风险偏好、有没有绝对不能接受的底线

## Step 1：信息层——建立事实池（Perplexity）

**Claude做什么**：根据用户情况生成可直接粘贴进Perplexity的Prompt。

向用户说明：
> "第一步去 Perplexity 建立行业事实池。它的优势是：每条结论附带可追溯的数据来源，不编造。把下面这个 Prompt 直接粘贴进去："

### 企业决策 Prompt（根据用户信息填入变量）：

```
过去三年（2022-2025）[行业名称]行业：
1）市场规模与增速区间（按年）
2）毛利率/净利率常见区间（如有）
3）客户采购预算变化与回款周期趋势
4）并购/整合趋势与典型案例（2023-2025）
5）AI对该行业的影响：产品形态、定价模式、交付成本、竞争格局

要求：每条结论给出可追溯数据来源（机构/报告/新闻出处），按要点输出，不要写长文。
```

### 个人决策 Prompt：

```
关于[赛道/行业/方向]，请提供：
1）目前这个领域的市场规模与增速（2022-2025）
2）从业者/创作者的收入区间与变现模式（按发展阶段）
3）典型成功案例与失败案例（附来源）
4）AI/技术变化对这个领域的影响趋势
5）入门门槛与核心竞争壁垒

要求：每条结论给出可追溯来源，按要点输出，不要写长文。
```

用户拿到Perplexity结果后，把结果发回Claude，进入Step 2。

## Step 2：结构层——识别"已知的未知"（Claude 亲自执行）⭐

**这是Claude的主场。** 用户把Perplexity结果 + 自己的数据一起给Claude，Claude做结构化梳理。

Claude的任务：
1. 将所有信息按维度归类，只保留硬核事实，剔除主观描述（"客户反馈还不错"、"团队挺努力"等一律剔除）
2. 把散装信息变成思维骨架——建立坐标系
3. 找出"不知道自己不知道什么"——输出事实缺口清单

### 企业决策输出格式：

```
【A. 财务与现金流事实】
- 收入：X → 增速从X%降至X%
- 毛利率：X%（行业均值X%，差距Xpp）
- 净利润：-X万，现金流：-X万
- 研发投入：X%营收

【B. 增长与客户结构事实】
- 客户数：X家，客单价：X万/年
- 续费率：X%（健康值通常>85%）
- 前X大客户占X%（集中度：高/中/低）
- 新签增速：X% vs 行业增速：X%

【C. 成本与组织事实】
- 人力成本占比：X%（固定成本压力：高/中/低）
- 总成本X万 vs 营收X万 → 盈亏平衡点在X

【D. 行业趋势映射事实】
（将Perplexity数据与内部数据对比，列影响路径，不下结论）
- 趋势①可能如何影响该业务：...
- 趋势②可能如何影响该业务：...

【事实缺口清单】（还缺以下信息才能更严谨决策）
1. ...
2. ...
（最多10条，按重要性排序）
```

### 个人决策输出格式：

```
【A. 现状与资源事实】
- 当前收入/可用时间/财务缓冲窗口：...

【B. 目标与约束事实】
- 核心目标排序：...
- 底线约束：...

【C. 外部趋势映射事实】
- 趋势①对这个选择的影响路径：...
- 趋势②对这个选择的影响路径：...

【事实缺口清单】
1. ...
（最多8条）
```

完成后告诉用户：
> "结构化梳理完成。你看一下有没有失真的地方，或补充信息？确认后我把这份结构打包成 GPT 的推演 Prompt。"

## Step 3：推理层——三路径博弈（GPT）

**Claude做什么**：把Step 2的结构化输出打包成可直接粘贴进GPT的Prompt。

向用户说明：
> "第三步去 GPT 做路径推演。GPT 在复杂博弈和逻辑发散上更出色。我们不问它'该怎么办'，而是让它展开三条路的风险地图。把 Step 2 的结构化结果 + 下面这个 Prompt 一起粘贴给 GPT："

### 企业决策 Prompt：

```
你是企业战略与经营分析顾问。
基于以下【事实整理】与【行业趋势】，请做"路径博弈推演"，不要给唯一答案。

【粘贴Step 2输出的结构化事实】

决策问题：[用户的决策问题]
路径：
1）立即裁撤/退出
2）缩减规模保留核心团队
3）继续投资并进行转型升级

对每条路径输出（必须分点）：
- 12个月现金流影响（用区间/方向/关键变量表达；需假设请写清楚）
- 2年后业务可能状态（给2种情景：Base/Downside 或 Base/Upside）
- 关键风险点（按：财务/客户/组织/声誉分类）
- 对公司整体战略影响
- 该路径成立的假设前提（最多8条）

最后输出：三路径的"可逆性/不可逆性"对比（哪些决策做了就很难回头？）
```

### 个人决策 Prompt：

```
你是职业发展与人生决策顾问。
基于以下【个人情况整理】与【行业趋势】，请做"路径博弈推演"，不要给唯一答案。

【粘贴Step 2输出的结构化事实】

决策问题：[用户的决策问题]
路径：
1）[路径A，如：继续现有工作]
2）[路径B，如：主业+副业并行]
3）[路径C，如：全身投入新方向]

对每条路径输出：
- 12个月财务与时间影响
- 2年后可能状态（Base/Downside两种）
- 关键风险点（按：财务/机会成本/精力/外部环境分类）
- 对长期目标的影响
- 该路径成立的假设前提（最多6条）

最后输出：三路径的"可逆性/不可逆性"对比
```

用户拿到GPT结果后发回Claude，进入Step 4。

## Step 4：系统层——决策工程化（GPT → Notion）

**Claude做什么**：生成两个Prompt，让用户依次在GPT里执行。

向用户说明：
> "第四步分两个动作：先压缩成一屏对比表，再生成 Notion 可导入的 CSV。这步的核心不是算分，而是把这次决策变成**可迭代的系统**——数据变了、假设变了，直接在 Notion 里更新；半年后复盘能看到当时基于什么权重做的决定。"

### 4.1 先发给GPT：压缩为一屏对比表

```
将以上三种路径推演压缩为"一屏可读"的对比表。
表格列固定为：
Path | 12M现金流 | 2Y状态 | 关键风险 | 风险等级 | 战略价值 | 假设前提 | 可逆性

规则：
- 每个单元格不超过40字
- 用短语，不写段落
- 风险等级必须给"高/中/低"
```

### 4.2 再发给GPT：生成Notion可导入CSV

先告诉用户填入以下参数，Claude帮他拼好完整Prompt：

- DecisionID（如 DEC-2026-03-XXX-001）
- Owner（决策负责人姓名）
- DecisionDate（今天日期）
- 权重设定（不确定就用默认值）

**权重默认值说明**（引导用户选择）：

企业决策：现金流0.40 / 增长潜力0.30 / 战略价值0.20 / 风险-0.10
- 收缩期 → 现金流提至0.50
- 扩张期 → 增长提至0.40

个人决策：财务可持续0.35 / 成长潜力0.30 / 目标契合0.25 / 风险-0.10
- 经济压力大 → 财务可持续提至0.50
- 处于探索期 → 成长潜力提至0.40

**生成的CSV Prompt模板**：

```
请把三路径对比结果输出为 CSV（逗号分隔），满足 Notion Import：

1）第一行是表头
2）字段（按此顺序）：
DecisionID,Path,PathType,Owner,Status,DecisionDate,12M_Cashflow,2Y_State,Key_Risks,Risk_Level,Strategic_Value,Assumptions,Reversibility,Score_A,Score_B,Score_C,Score_D,Weight_A,Weight_B,Weight_C,Weight_D,Weighted_Total,Notes

3）字段类型：
- PathType/Status/Risk_Level = Select
- Score_*/Weight_* = Number
- DecisionDate = Date（YYYY-MM-DD）

4）Weighted_Total 直接算好填入
5）含逗号的字段用英文双引号包裹
6）只输出 CSV 代码块，不要解释

参数：
DecisionID=[X]
Owner=[X]
DecisionDate=[X]
Weight_A=[X], Weight_B=[X], Weight_C=[X], Weight_D=[X]
```

用户拿到CSV → Notion → New Database → Import CSV → 三条路径变成三条可管理的记录。

## Step 5：表达层——视觉化汇报（GPT → Gamma）

**使用场景**：需要向高管/董事会汇报，或对外呈现时。内部自用到Step 4即可。

向用户说明：
> "最后一步：把冰冷的数据变成有说服力的商业叙事。注意不能把 Notion 表格直接扔给 Gamma——决策逻辑是横向对比，汇报逻辑是纵向叙事。GPT 是'设计师'，Gamma 是'裁缝'——先让 GPT 定好汇报结构，Gamma 只做视觉呈现。
> 把 Notion 的评分结果 + 三路径对比表 + 下面这个 Prompt 给 GPT："

```
以下是本次决策的三路径对比摘要与评分排序结果：

【路径对比摘要】
（粘贴Step 4的一屏对比表）

【评分排序结果】
（粘贴三条路径的 Weighted_Total + 权重设定说明）

汇报对象：[高管团队/董事会/投资人]
决策目标：[用户的决策问题]
需要输出：明确建议 + 风险说明 + 下一步行动框架

请生成适合10页以内PPT的汇报逻辑大纲，结构包含：
1）背景与问题定义
2）关键事实摘要
3）三路径对比
4）评分逻辑与权重说明
5）建议方案与理由
6）风险控制与补充数据需求
7）行动计划与时间线

要求：逻辑递进，标题可直接作为PPT页标题，每页3-5条要点
```

用户拿到大纲 → 粘贴进 Gamma → 生成可发给CEO的决策方案PPT。

## 完整流程节奏

```
Claude收集信息 + 判断决策类型
        ↓
Step 1：Claude生成Prompt → 用户去Perplexity → 结果发回Claude
        ↓
Step 2：Claude亲自执行结构化分析 ⭐ → 用户确认
        ↓
Step 3：Claude生成Prompt → 用户去GPT → 结果发回Claude
        ↓
Step 4：Claude生成两个Prompt → 用户去GPT执行 → 导入Notion
        ↓
Step 5（可选）：Claude生成Prompt → 用户去GPT → 大纲带去Gamma
```

每步节奏：Claude给Prompt + 说明这步在做什么/为什么用这个工具 → 用户执行 → 发回结果 → Claude确认质量准备下一步。

---

## 核心原则

1. **不替用户做决定**：展现路径和代价，判断权在人
2. **假设透明化**：每条推演写清假设前提，用户自己判断假设是否成立
3. **高分路径≠最优选**：评分只是参考，关键是用户自己对假设前提的判断
4. **Gamma不参与思考**：它只负责让思考"看起来很贵"
